# VPN 및 원격 접속 트러블슈팅

## 1. VPN 접속 오류 해결 방법

### 1.1 VPN이 연결되지 않는 경우

가장 흔한 원인 및 조치:

**원인 1: VPN 클라이언트 미설치 또는 구버전**
1. IT 포털(it.company.com)에서 최신 GlobalProtect VPN 클라이언트 다운로드
2. 기존 클라이언트 제거 후 재설치
3. 설치 후 PC 재부팅

**원인 2: 아이디/비밀번호 오류**
1. 사내 계정(사번@company.com)과 동일한 비밀번호 사용
2. 대소문자 확인
3. 비밀번호 만료 확인 (90일마다 변경 필요)
4. 비밀번호 분실 시: IT헬프데스크(내선: 5000) 또는 it-help@company.com

**원인 3: MFA(다중 인증) 문제**
1. 인증 앱(Microsoft Authenticator) 재실행
2. 앱 알림이 오지 않는 경우: 앱 → 계정 → 새로고침
3. 앱 삭제 후 재설치 필요 시 IT헬프데스크 연락

**원인 4: 방화벽/보안 소프트웨어 차단**
1. Windows Defender 또는 설치된 백신 프로그램 확인
2. VPN 포트(UDP 4501, TCP 443) 차단 여부 확인
3. 공공 와이파이 사용 시 VPN 포트 차단 가능성 있음

### 1.2 VPN 연결됐지만 사내 자원 접근 불가

1. VPN 연결 상태 아이콘 확인 (GlobalProtect 트레이 아이콘)
2. 사내 IP 주소 할당 여부 확인: ipconfig /all (Windows), ifconfig (Mac)
   - 정상: 10.10.x.x 대역 IP 할당
3. DNS 설정 확인: 8.8.8.8 → 10.10.1.10으로 변경
4. VPN 연결 해제 후 재연결

### 1.3 Mac에서 VPN 접속 오류

1. GlobalProtect 앱 → "설정" → 연결 서버: vpn.company.com 확인
2. 시스템 환경설정 → 개인 정보 보호 및 보안 → GlobalProtect 허용
3. Mac OS 업데이트 후 발생 시: 앱 삭제 후 재설치

## 2. VPN 사용 규칙

- 재택/외부 근무 시 반드시 VPN 접속 후 사내 시스템 접근
- VPN 접속 유지 시간 제한: 최대 8시간 (이후 자동 해제)
- 개인 용도 트래픽은 VPN을 통하지 않음 (Split Tunnel 적용)
- 접속 이력은 보안팀에 의해 모니터링됨

## 3. 원격 데스크탑 (RDP) 접속 문제

### 3.1 원격 데스크탑 연결 실패
1. VPN 먼저 연결 후 RDP 시도
2. 접속 PC의 전원이 켜져 있는지 확인 (동료에게 요청)
3. IP 주소 확인: 사내 IT 포털에서 자신의 PC IP 확인
4. Windows 원격 데스크탑 허용 설정 확인 (시스템 → 원격 설정)
5. 문제 지속 시 IT헬프데스크 연락 (내선: 5000)
