# 전사 업무 지원 RAG 챗봇 구현 보고서

---

## 1. 구현 결과 요약

README.md에 정의된 모든 요구사항을 구현 완료하였다. 5개 업무 도메인(현장 매뉴얼, 사내 규정/HR, IT 헬프데스크, 영업/고객 대응, 교육/온보딩)에 대한 RAG 기반 챗봇이 정상 동작하며, Docker Compose 환경에서 실행 가능한 서비스로 구성하였다.

---

## 3. 베이스 코드 대비 변경/추가 사항

### 3.1 LangGraph 파이프라인 구현 (`backend/app/graph.py`)

**변경 내용**: 단순 LLM 호출에서 **3-노드 LangGraph 상태 그래프**로 전환

- `ChatState` TypedDict에 `domain`, `context`, `sources` 필드 추가하여 노드 간 상태 전달
- `router_node` → `retriever_node` → `generator_node` 순차 파이프라인 구성
- 멀티모달 메시지(이미지 포함 리스트 타입) 파싱 로직(`_get_last_human_text`) 추가
- 도메인별 전용 System Prompt 템플릿으로 답변 품질 향상

```
[변경 전] User → LLM → 단일 응답
[변경 후] User → router_node → retriever_node → generator_node → 구조화된 응답
```

---

### 3.2 LLM 기반 도메인 라우터 신규 구현 (`backend/app/router.py`)

**신규 구현**: 베이스 코드에 없던 도메인 라우팅 모듈 전체 작성

- 5개 도메인의 특성을 정의한 `DOMAIN_DESCRIPTIONS` 딕셔너리
- LLM이 질문을 받아 도메인 키 하나만 출력하도록 강제하는 System Prompt 설계
- LLM 응답이 유효하지 않을 경우 한국어 키워드 매칭으로 폴백(`_keyword_fallback`)

---

### 3.3 멀티 도메인 Chroma 벡터 DB 구현 (`backend/app/rag.py`)

**변경 내용**: 단일 도메인 RAG에서 **5개 독립 도메인 컬렉션** 구조로 확장

- 도메인별 Chroma 컬렉션(`rag_manual`, `rag_hr`, `rag_it`, `rag_sales`, `rag_education`) 관리
- 메모리 캐시(`_retrievers` 딕셔너리)로 중복 벡터 DB 로드 방지
- 지식 문서 자동 로드·청킹(chunk_size=500, overlap=80) 및 벡터 색인 파이프라인
- 컬렉션이 없으면 자동 생성, 있으면 기존 컬렉션 재활용하는 로직

| 참고 문서 | `st.expander`로 접기/펼치기 가능한 출처 파일명 목록 표시 |
| 이미지 업로드 | 사이드바에 이미지 업로더 추가, Base64 인코딩 후 백엔드 전송 |
| 세션 관리 | UUID 기반 Thread ID로 대화 이력 유지, 새 대화 시작 버튼 |
| 사이드바 | 지원 도메인 목록, 세션 정보, 백엔드 URL 표시 |

### 3.6 Docker Compose 구성 개선 (`docker-compose.yml`)

**변경 내용**:

- 백엔드 **헬스 체크** 추가(`/health` 엔드포인트, 30초 간격)
